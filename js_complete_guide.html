JavaScript Fundamentals: Complete Guide

Learn the core concepts of JavaScript programming from variables to functions

Introduction to JavaScript

JavaScript is a versatile programming language that powers interactive behavior on websites. It allows you to create dynamic content, control multimedia, animate images, and much more.

Key Characteristics of JavaScript:

  • High-level, interpreted programming language
  • Dynamic typing
  • Prototype-based object orientation
  • First-class functions
  • Runs on the client-side and server-side (with Node.js)

Your First JavaScript Code

// Simple JavaScript example
console.log("Hello, World!");

// Display alert
alert("Welcome to JavaScript!");

Where to Write JavaScript

JavaScript can be added to HTML in three ways:

  1. Inline in HTML elements
  2. In <script> tags in the head or body
  3. In external .js files

Variables and Data Types

Variables are used to store information that can be referenced and manipulated in a program. JavaScript has three ways to declare variables: var, let, and const.

Variable Declaration: Modern Best Practices

While JavaScript supports var, let, and const, modern best practices recommend:

  • Use const by default for variables that won't be reassigned
  • Use let when you need to reassign values
  • Avoid var due to its function-scoping behavior and hoisting issues
// Using const (block-scoped, cannot be reassigned)
const PI = 3.14159;
const API_URL = "https://api.example.com";

// Using let (block-scoped, can be reassigned)
let counter = 0;
let userName = "John";

// Using var (function-scoped, avoid in modern JS)
var oldVariable = "Avoid using var";

Scope: Function vs Block

Function Scope: Variables declared with var are function-scoped, meaning they are accessible anywhere within the function they are declared in.

Block Scope: Variables declared with let and const are block-scoped, meaning they are only accessible within the block (enclosed by {}) they are declared in.

// Function scope example
function functionScopeExample() {
    if (true) {
        var functionScoped = "I'm accessible anywhere in this function";
    }
    console.log(functionScoped); // Works fine
}

// Block scope example
function blockScopeExample() {
    if (true) {
        let blockScoped = "I'm only accessible in this block";
        const alsoBlockScoped = "Me too!";
    }
    console.log(blockScoped); // Error: blockScoped is not defined
}

JavaScript Data Types

JavaScript has dynamic types, meaning the same variable can be used to hold different data types. The language has primitive and object data types.

String

Represents textual data. Enclosed in quotes (single or double).

let name = "John";
let message = 'Hello world!';

Number

Represents both integer and floating-point numbers.

let integer = 42;
let decimal = 3.14;
let negative = -15;

Boolean

Represents a logical entity with two values: true or false.

let isActive = true;
let isCompleted = false;

Null

Represents the intentional absence of any object value.

let emptyValue = null;

Undefined

A variable that has been declared but not assigned a value.

let notAssigned;
console.log(notAssigned); // undefined

BigInt

Represents integers larger than 2^53 - 1. Created by appending n to the end of an integer.

let bigNumber = 123456789012345678901234567890n;

Symbol

A unique and immutable primitive value that may be used as a property key.

let sym = Symbol('description');
let sym2 = Symbol('description');
console.log(sym === sym2); // false

Object

Collections of key-value pairs used to store complex entities.

let person = {
    name: "Alice",
    age: 30,
    isStudent: false
};

Array

Ordered collections of values, accessed by numerical index.

let colors = ["red", "green", "blue"];
let numbers = [1, 2, 3, 4, 5];

Operators and Expressions

Operators are used to perform operations on variables and values. JavaScript includes various types of operators for different purposes.

Types of Operators

  • Arithmetic Operators: +, -, *, /, %, ** (exponentiation)
  • Assignment Operators: =, +=, -=, *=, /=, %=
  • Comparison Operators: ==, ===, !=, !==, >, <, >=, <=
  • Logical Operators: &&, ||, !
  • Ternary Operator: condition ? expr1 : expr2

Arithmetic Operations

let a = 10;
let b = 5;

console.log(a + b); // 15
console.log(a - b); // 5
console.log(a * b); // 50
console.log(a / b); // 2
console.log(a % b); // 0
console.log(a ** b); // 100000

Comparison Operations

let x = 5;
let y = "5";

console.log(x == y); // true (value equality)
console.log(x === y); // false (strict equality)
console.log(x != y); // false
console.log(x !== y); // true
console.log(x > 3); // true
console.log(x < 10); // true

Logical Operators

Logical operators are used to combine or manipulate boolean values.

AND Operator (&&)

Returns true if both operands are true.

let hasAccount = true;
let isLoggedIn = false;

console.log(hasAccount && isLoggedIn); // false

let age = 25;
let hasLicense = true;
console.log(age >= 18 && hasLicense); // true

OR Operator (||)

Returns true if at least one operand is true.

let hasCreditCard = true;
let hasPayPal = false;

console.log(hasCreditCard || hasPayPal); // true

let isWeekend = false;
let isHoliday = true;
console.log(isWeekend || isHoliday); // true

NOT Operator (!)

Returns the opposite boolean value of its operand.

let isLoggedIn = false;

console.log(!isLoggedIn); // true

let isRaining = true;
console.log(!isRaining); // false

Ternary Operator

The ternary operator is a concise way to write conditional statements.

Syntax: condition ? expressionIfTrue : expressionIfFalse

// Traditional if-else statement
let message;
let age = 20;

if (age >= 18) {
    message = "You are an adult";
} else {
    message = "You are a minor";
}

// Equivalent ternary operator
let message = age >= 18 ? "You are an adult" : "You are a minor";

// More complex example
let score = 85;
let grade = score >= 90 ? "A" :
            score >= 80 ? "B" :
            score >= 70 ? "C" :
            score >= 60 ? "D" : "F";
            
console.log(grade); // "B"

Control Flow

Control flow statements allow you to control the execution of your code based on conditions and loops.

Conditional Statements

// if statement
if (condition) {
  // code to execute if condition is true
}

// if-else statement
if (condition) {
  // code if true
} else {
  // code if false
}

// else-if statement
if (condition1) {
  // code if condition1 is true
} else if (condition2) {
  // code if condition2 is true
} else {
  // code if all conditions are false
}

// switch statement
switch(expression) {
  case value1:
    // code to execute when expression matches value1
    break;
  case value2:
    // code to execute when expression matches value2
    break;
  default:
    // code to execute if no cases match
}

Looping Statements

// for loop - when you know how many times to iterate
for (let i = 0; i < 5; i++) {
  // code to repeat 5 times
}

// while loop - when you don't know how many times to iterate
while (condition) {
  // code to repeat while condition is true
}

// do-while loop - executes at least once
do {
  // code to execute at least once
} while (condition);

Switch Statement Explained

The switch statement evaluates an expression and executes code based on matching cases.

let day = 3;
let dayName;

switch (day) {
  case 1:
    dayName = "Monday";
    break;
  case 2:
    dayName = "Tuesday";
    break;
  case 3:
    dayName = "Wednesday";
    break;
  case 4:
    dayName = "Thursday";
    break;
  case 5:
    dayName = "Friday";
    break;
  case 6:
    dayName = "Saturday";
    break;
  case 7:
    dayName = "Sunday";
    break;
  default:
    dayName = "Invalid day";
}

console.log(dayName); // "Wednesday"

Important: The break statement is crucial to prevent "fall-through" where multiple cases execute. The default case runs when no other cases match.

Loop Differences

For Loop

Best when you know how many iterations you need.

// Print numbers 1 to 5
for (let i = 1; i <= 5; i++) {
  console.log(i);
}

While Loop

Best when you need to loop until a condition changes.

// Count down from 5
let count = 5;
while (count > 0) {
  console.log(count);
  count--;
}

Do-While Loop

Guarantees at least one execution, even if condition is false.

// Ask for input at least once
let userInput;
do {
  userInput = prompt("Enter yes or no:");
} while (userInput !== "yes" && userInput !== "no");

Functions

Functions are one of the fundamental building blocks in JavaScript. They allow you to package up code for reuse throughout your program.

Function Declarations vs Expressions

JavaScript offers multiple ways to define functions, each with different characteristics:

// Function declaration (hoisted)
function add(a, b) {
  return a + b;
}

// Function expression (not hoisted)
const multiply = function(a, b) {
  return a * b;
};

// Arrow function (ES6, concise syntax, no own 'this')
const divide = (a, b) => {
  return a / b;
};

// Concise arrow function (implicit return)
const square = x => x * x;

Function Parameters and Arguments

Functions can accept parameters (listed in the function definition) and arguments (the actual values passed to the function).

// Parameters with default values
function greet(name = "Guest", greeting = "Hello") {
  return `${greeting}, ${name}!`;
}

console.log(greet()); // "Hello, Guest!"
console.log(greet("Alice")); // "Hello, Alice!"
console.log(greet("Bob", "Hi")); // "Hi, Bob!"

// Rest parameters (collect all remaining arguments)
function sum(...numbers) {
  return numbers.reduce((total, num) => total + num, 0);
}

console.log(sum(1, 2, 3, 4, 5)); // 15

Return Values

Functions can return values using the return statement. If no return is specified, the function returns undefined.

// Explicit return
function getFullName(firstName, lastName) {
  return `${firstName} ${lastName}`;
}

// Implicit return (arrow functions)
const getSquare = x => x * x;

// No return (returns undefined)
function logMessage(message) {
  console.log(message);
  // No return statement
}

let result = logMessage("Hello"); // result is undefined

Higher-Order Functions

Functions that operate on other functions, either by taking them as arguments or by returning them.

// Function that takes a function as argument
function repeatOperation(operation, times) {
  for (let i = 0; i < times; i++) {
    operation(i);
  }
}

repeatOperation(i => console.log(`Iteration ${i}`), 3);

// Function that returns a function
function createMultiplier(multiplier) {
  return function(x) {
    return x * multiplier;
  };
}

const double = createMultiplier(2);
console.log(double(5)); // 10

Arrays and Objects

Arrays and objects are complex data types that allow you to store collections of data.

Arrays

Arrays are used to store multiple values in a single variable.

// Creating arrays
let fruits = ["Apple", "Banana", "Orange"];
let numbers = [1, 2, 3, 4, 5];

// Accessing array elements
console.log(fruits[0]); // "Apple"
console.log(fruits.length); // 3

// Array methods
fruits.push("Mango"); // adds to the end
fruits.pop(); // removes from the end
fruits.unshift("Strawberry"); // adds to the beginning
fruits.shift(); // removes from the beginning

Array.slice() vs Array.splice()

These are two important array methods with different purposes:

// slice() returns a shallow copy of a portion of an array
let fruits = ["Apple", "Banana", "Orange", "Mango", "Lemon"];
let citrus = fruits.slice(1, 3); // ["Banana", "Orange"]
let lastTwo = fruits.slice(-2); // ["Mango", "Lemon"]

// splice() changes the contents of an array by removing or replacing elements
let months = ["Jan", "March", "April", "June"];
months.splice(1, 0, "Feb"); // Inserts at index 1
// Result: ["Jan", "Feb", "March", "April", "June"]

months.splice(4, 1, "May"); // Replaces 1 element at index 4
// Result: ["Jan", "Feb", "March", "April", "May"]

Objects

Objects are collections of key-value pairs.

// Creating objects
let person = {
  firstName: "John",
  lastName: "Doe",
  age: 30,
  isStudent: false
};

// Accessing object properties
console.log(person.firstName); // "John"
console.log(person["lastName"]); // "Doe"

// Adding properties
person.email = "john@example.com";

Array Methods

let numbers = [1, 2, 3, 4, 5];

// map - creates new array
let doubled = numbers.map(n => n * 2);

// filter - creates filtered array
let evens = numbers.filter(n => n % 2 === 0);

// reduce - reduces to a single value
let sum = numbers.reduce((total, n) => total + n, 0);

console.log(doubled); // [2, 4, 6, 8, 10]
console.log(evens); // [2, 4]
console.log(sum); // 15

Object Operations

let car = {
  make: "Toyota",
  model: "Camry",
  year: 2020
};

// Getting keys
let keys = Object.keys(car); // ["make", "model", "year"]

// Getting values
let values = Object.values(car); // ["Toyota", "Camry", 2020]

// Checking if property exists
console.log("make" in car); // true
console.log(car.hasOwnProperty("model")); // true

// Your code here
function isPrime(number) {
  if (number <= 1) return false;
  if (number === 2) return true;
  
  for (let i = 2; i <= Math.sqrt(number); i++) {
    if (number % i === 0) {
      return false;
    }
  }
  return true;
}

// Example usage:

console.log(isPrime(10)); // false

Advanced JavaScript Concepts

Math Object Methods

JavaScript's Math object provides mathematical constants and functions.

Math.floor()

Returns the largest integer less than or equal to a number.

console.log(Math.floor(4.7)); // 4
console.log(Math.floor(8.1)); // 8
console.log(Math.floor(-3.2)); // -4

Math.ceil()

Returns the smallest integer greater than or equal to a number.

console.log(Math.ceil(4.2)); // 5
console.log(Math.ceil(8.9)); // 9
console.log(Math.ceil(-3.7)); // -3

Math.round()

Returns the value of a number rounded to the nearest integer.

console.log(Math.round(4.5)); // 5
console.log(Math.round(4.4)); // 4
console.log(Math.round(-3.7)); // -4

Array.map() Explained

The map() method creates a new array with the results of calling a function on every element in the array.

let numbers = [1, 2, 3, 4, 5];

// Double each number
let doubled = numbers.map(function(num) {
  return num * 2;
});
// doubled is [2, 4, 6, 8, 10]

// Using arrow function
let squared = numbers.map(num => num * num);
// squared is [1, 4, 9, 16, 25]

// Working with objects
let users = [
  { name: "Alice", age: 30 },
  { name: "Bob", age: 25 },
  { name: "Charlie", age: 35 }
];

let names = users.map(user => user.name);
// names is ["Alice", "Bob", "Charlie"]

Important: map() does not change the original array, it returns a new array.

JavaScript Versions

  • ES5 (2009) - Added strict mode, JSON support
  • ES6 (2015) - let/const, arrow functions, classes
  • ES2016 - Array.includes(), exponentiation operator
  • ES2017 - Async functions, shared memory
  • ES2018 - Rest/spread properties, Promise.finally()
  • ES2019 - Array.flat(), Object.fromEntries()
  • ES2020 - BigInt, nullish coalescing, optional chaining

Best Practices

  • Use const by default, let when needed
  • Use strict equality (===) instead of (==)
  • Use descriptive variable and function names
  • Keep functions small and focused
  • Always declare variables before use
  • Avoid using var in modern JavaScript

Common Pitfalls

  • Hoisting issues with var declarations
  • Confusion between == and ===
  • Forgetting to use "new" with constructors
  • Modifying objects while iterating over them
  • Scope issues in callbacks and closures

Debugging Tips

  • Use console.log() for basic debugging
  • Leverage browser developer tools
  • Use debugger statement for breakpoints
  • Check for type errors with typeof
  • Handle errors with try-catch blocks

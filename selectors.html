CSS Selectors Explained

Understanding CSS selectors including combinators and pseudo-selectors

Basic Selectors

CSS selectors are patterns used to select the element(s) you want to style. Let's start with the basic selectors:

Selector Example Description
Element p Selects all <p> elements
Class .intro Selects all elements with class="intro"
ID #header Selects the element with id="header"
Universal * Selects all elements
Attribute [target] Selects all elements with a target attribute

Combinator Selectors

Combinator selectors combine other selectors to target elements based on their relationship to other elements.

Descendant Selector (space)

Matches all elements that are descendants of a specified element.

/* Selects all .child elements inside .parent */
.parent .child {
    background-color: #ffe6cc;
}
Parent element
Child element (selected)
Nested child (selected)

Child Selector (>)

Matches all elements that are direct children of a specified element.

/* Selects only direct .direct-child elements of .parent */
.parent > .direct-child {
    background-color: #d4edda;
}
Parent element
Direct child (selected)
Nested direct child (not selected)

Adjacent Sibling Selector (+)

Matches an element that is immediately after another specific element.

/* Selects .adjacent-sibling immediately after .sibling */
.sibling + .adjacent-sibling {
    background-color: #f8d7da;
}
Sibling element
Adjacent sibling (selected)
Other element
Another adjacent sibling (not selected)

General Sibling Selector (~)

Matches all elements that are next siblings of a specified element.

/* Selects all .general-sibling elements after .sibling */
.sibling ~ .general-sibling {
    background-color: #cce5ff;
}
Sibling element
General sibling (selected)
Other element
Another general sibling (selected)

Pseudo-class Selectors (:)

Pseudo-classes select elements based on their state or position, not just their name or attributes.

User Action States

Hover over me!

Click and hold to see active state

Structural Positions

  • First child (bold)
  • Even item (pink)
  • Odd item (green)
  • Even item (pink)
  • Last child (italic)

Common Pseudo-classes

/* User action states */
a:hover { }        /* When mouse over element */
a:active { }       /* When element is being activated */
input:focus { }    /* When element has focus */

/* Structural pseudo-classes */
li:first-child { } /* First child of its parent */
li:last-child { }  /* Last child of its parent */
li:nth-child(2) { }/* Second child of its parent */
li:nth-child(2n) { }/* Every even child */

/* Form states */
input:checked { }  /* Checked radio buttons or checkboxes */
input:disabled { } /* Disabled form elements */
input:required { } /* Required form elements */

Pseudo-element Selectors (::)

Pseudo-elements style specific parts of an element rather than the element itself.

::before and ::after

This content has pseudo-elements

::first-line and ::first-letter

This is a paragraph that demonstrates the ::first-line and ::first-letter pseudo-elements. The first line will be styled differently, and the first letter will be larger than the rest.

Common Pseudo-elements

/* Insert content before an element */
.element::before {
    content: "Before";
}

/* Insert content after an element */
.element::after {
    content: "After";
}

/* Style the first line of text */
p::first-line {
    font-weight: bold;
}

/* Style the first letter of text */
p::first-letter {
    font-size: 2em;
    color: red;
}

/* Style user selection */
::selection {
    background: yellow;
    color: black;
}

Key Differences

  • Pseudo-classes (:) select elements based on state
  • Pseudo-elements (::) style specific parts of elements
  • Modern browsers support both : and :: for pseudo-elements, but :: is preferred

Selector Specificity

When multiple selectors target the same element, specificity determines which styles apply:

  1. Inline styles (1000 points)
  2. IDs (100 points)
  3. Classes, attributes, pseudo-classes (10 points)
  4. Elements, pseudo-elements (1 point)

Higher specificity overrides lower specificity.

Browser Support

Most modern selectors are supported in all current browsers:

  • Basic selectors: All browsers
  • Combinator selectors: All browsers
  • Pseudo-classes: Good support in modern browsers
  • Pseudo-elements: Good support with :: syntax in modern browsers

Best Practices

  • Use classes for reusable styles
  • Avoid overly specific selectors
  • Use pseudo-classes for interactive states
  • Use pseudo-elements for decorative elements
  • Test selectors across browsers

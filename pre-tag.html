Understanding the <pre> HTML Tag

Preserving formatting and displaying code snippets effectively

What is the <pre> Tag?

The <pre> tag in HTML defines preformatted text. Text within a <pre> element is displayed in a fixed-width font (usually Courier), and it preserves both spaces and line breaks exactly as they appear in the HTML code.

This is particularly useful for displaying:

  • Code snippets
  • ASCII art
  • Text-based diagrams
  • Formatted text where spacing matters
  • Keyboard input/output examples

Code Examples

Basic Usage

Here's a simple example of using the <pre> tag:

HTML Code:

<pre>
    This text will
    preserve all
    spaces     and
    line breaks.
</pre>

Output:

    This text will
    preserve all
    spaces     and
    line breaks.

Displaying Code Snippets

The <pre> tag is commonly used with the <code> tag to display code snippets:

HTML Code:

<pre><code>
function greet() {
    console.log("Hey, GabbyTech!");
}
</code></pre>

Output:

function greet() {
    console.log("Hey, GabbyTech!");
}

Key Characteristics

  • Preserves whitespace and line breaks
  • Uses a monospaced font by default
  • Has default margins in most browsers
  • Can be styled with CSS
  • Often used with <code> for code blocks

Best Practices

  • Use for code snippets and preformatted text
  • Combine with <code> for semantic markup
  • Add overflow handling for long lines
  • Style with CSS to improve readability
  • Consider accessibility with screen readers

CSS Styling Tips

pre {
    background: #f4f4f4;
    padding: 15px;
    border-radius: 5px;
    overflow-x: auto;
    font-family: 'Courier New', monospace;
    line-height: 1.5;
}

Browser Support

What are CSS Animations and Transitions?

CSS Animations and Transitions allow you to create motion and visual feedback on web pages without JavaScript. They provide smooth, performant ways to enhance user experience and draw attention to important elements.

CSS Transitions provide a way to control animation speed when changing CSS properties. Instead of having property changes take effect immediately, you can cause the changes to occur over a period of time.

CSS Animations allow you to create complex animations by defining keyframes that specify the style values at various points during the animation sequence.

CSS Transitions

Transitions enable you to define the transition between two states of an element. Different states may be defined using pseudo-classes like :hover or :active, or by dynamically setting classes with JavaScript.

Transition Properties

CSS transitions are controlled using these properties:

  • transition-property: Specifies the name of the CSS property to apply the transition to
  • transition-duration: Sets the length of time a transition animation should take
  • transition-timing-function: Determines how intermediate values are calculated
  • transition-delay: Defines how long to wait before starting the transition
.element {
  transition: all 0.3s ease-in-out 0.1s;
  
  /* Shorthand: property | duration | timing-function | delay */
}

CSS Animations

CSS animations make it possible to animate transitions from one CSS style configuration to another. Animations consist of two components: a style describing the CSS animation and a set of keyframes that indicate the start and end states of the animation's style.

Animation Properties

CSS animations are controlled using these properties:

  • animation-name: Specifies the name of the @keyframes rule
  • animation-duration: Sets the length of time an animation takes
  • animation-timing-function: Determines how the animation progresses through keyframes
  • animation-delay: Defines how long to wait before starting the animation
  • animation-iteration-count: Sets the number of times an animation should run
  • animation-direction: Sets whether an animation should play forward, reverse, or alternate
  • animation-fill-mode: Sets how styles are applied before and after animation execution
  • animation-play-state: Allows pausing and resuming the animation
.element {
  animation: slide-in 1s ease-in-out 0.5s infinite alternate;
  
  /* Shorthand: name | duration | timing-function | delay | 
  iteration-count | direction | fill-mode | play-state */
}

Keyframes: The Heart of CSS Animations

The @keyframes rule is where you define what the animation should do at various points during its sequence. You specify keyframes using percentages, with 0% being the start of the animation and 100% being the end.

Keyframes Syntax

@keyframes animation-name {
  0% {
    /* CSS properties at start */
  }
  50% {
    /* CSS properties halfway */
  }
  100% {
    /* CSS properties at end */
  }
}

You can also use the keywords from (equivalent to 0%) and to (equivalent to 100%).

Simple Keyframes Example

Pulse Effect
@keyframes pulse {
  0% { transform: scale(1); }
  50% { transform: scale(1.1); }
  100% { transform: scale(1); }
}

Complex Keyframes Example

Spinning
@keyframes spin {
  0% { 
    transform: rotate(0deg); 
    background-color: #4a86e8;
  }
  50% { 
    transform: rotate(180deg); 
    background-color: #b21f1f;
  }
  100% { 
    transform: rotate(360deg); 
    background-color: #4a86e8;
  }
}

Practical Animation Examples

Let's look at some real-world examples of how animations and transitions can enhance user experience.

Loading Spinner

Create a simple loading spinner with CSS animations:

.spinner {
  width: 40px;
  height: 40px;
  border: 4px solid #f3f3f3;
  border-top: 4px solid #4a86e8;
  border-radius: 50%;
  animation: spin 1s linear infinite;
}

@keyframes spin {
  0% { transform: rotate(0deg); }
  100% { transform: rotate(360deg); }
}

Hover Effects for Buttons

.button {
  padding: 12px 24px;
  background: #4a86e8;
  color: white;
  border: none;
  border-radius: 4px;
  cursor: pointer;
  transition: all 0.3s ease;
}

.button:hover {
  background: #3a76d8;
  transform: translateY(-2px);
  box-shadow: 0 4px 8px rgba(0,0,0,0.1);
}

Fade-in Page Elements

Faded In Content
.fade-in-element {
  opacity: 0;
  animation: fadeIn 2s forwards;
}

@keyframes fadeIn {
  to { opacity: 1; }
}

Benefits of CSS Animations

  • Better performance than JavaScript animations
  • Smooth, hardware-accelerated animations
  • Simple syntax for common animation patterns
  • Works even when JavaScript is disabled
  • Easy to maintain and modify

Browser Support

CSS animations and transitions are supported in all modern browsers:

  • Chrome: 43+
  • Firefox: 16+
  • Safari: 9+
  • Edge: 12+
  • Opera: 30+

For older browsers, consider using JavaScript fallbacks or progressive enhancement.

Best Practices

  • Use transitions for simple state changes
  • Use animations for complex, multi-step sequences
  • Animate only transform and opacity properties for best performance
  • Keep animations subtle and purposeful
  • Provide controls for users who prefer reduced motion

Common Use Cases

  • Loading indicators and spinners
  • Hover effects on buttons and links
  • Page transition animations
  • Notification alerts
  • Form validation feedback
  • Interactive navigation elements

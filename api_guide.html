What is an API?

An API (Application Programming Interface) is a set of rules that allows different applications to communicate with each other. APIs enable information sharing and functionality access between software components.

Key Concepts:

  • APIs define methods and data formats for communication
  • They enable access to databases, services, and external functionality
  • We use APIs daily (social media logins, weather apps, form submissions)
  • Front-end integration means retrieving and displaying data from servers

RESTful APIs are the most common type you'll encounter as a front-end developer. They use standard HTTP methods and typically return data in JSON format.

API Types Comparison

REST API Characteristics

  • Uses HTTP methods (GET, POST, PUT, DELETE)
  • Stateless - each request contains all necessary information
  • Returns data typically in JSON format
  • Uses endpoints for specific resources

Third-party SDKs

  • Provide pre-built functionality (e.g., Facebook Login)
  • Often include UI components
  • Simplify complex integration processes
  • May have additional dependencies

Understanding HTTP Methods

HTTP APIs use specific methods to perform operations on resources. These methods correspond to CRUD operations (Create, Read, Update, Delete).

HTTP Methods

// GET - Retrieve data from a specific resource
fetch('https://api.example.com/users')

// POST - Submit data to be processed
fetch('https://api.example.com/users', {
  method: 'POST',
  body: JSON.stringify({ name: 'John' })
})

// PUT - Update existing data
fetch('https://api.example.com/users/123', {
  method: 'PUT',
  body: JSON.stringify({ name: 'John Doe' })
})

// DELETE - Remove specific data
fetch('https://api.example.com/users/123', {
  method: 'DELETE'
})

Making API Requests with Fetch

The Fetch API provides a modern interface for making HTTP requests in JavaScript. It returns Promises, making it easier to work with asynchronous operations.

Basic Fetch Usage

// Basic GET request
fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Error:', error));

// Using async/await
async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error('Error:', error);
  }
}

Request Options

// POST request with headers and body
fetch('https://api.example.com/data', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer your-token-here'
  },
  body: JSON.stringify({
    name: 'John Doe',
    email: 'john@example.com'
  })
})
.then(response => response.json())
.then(data => console.log(data));

Working with API Responses

Understanding how to handle API responses is crucial for building robust applications. API responses include status codes, headers, and the response body.

HTTP Status Codes

// Common status codes
200 - OK: Request succeeded
201 - Created: Resource created successfully
400 - Bad Request: Invalid request
401 - Unauthorized: Authentication required
403 - Forbidden: Access not permitted
404 - Not Found: Resource doesn't exist
500 - Internal Server Error: Server error

Handling Responses

fetch('https://api.example.com/data')
  .then(response => {
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }
    return response.json();
  })
  .then(data => {
    // Process the data
    displayData(data);
  })
  .catch(error => {
    // Handle errors
    showError(error.message);
  });

Authentication & API Keys

Many APIs require authentication to control access and track usage. Common authentication methods include API keys, OAuth, and JWT tokens.

API Key Authentication

// Adding API key as a query parameter
fetch('https://api.example.com/data?api_key=your-api-key-here')
  .then(response => response.json())
  .then(data => console.log(data));

// Adding API key in headers
fetch('https://api.example.com/data', {
  headers: {
    'X-API-Key': 'your-api-key-here'
  }
})
.then(response => response.json())
.then(data => console.log(data));

Bearer Token Authentication

// Using Bearer token for authentication
fetch('https://api.example.com/data', {
  headers: {
    'Authorization': 'Bearer your-token-here'
  }
})
.then(response => response.json())
.then(data => console.log(data));

Error Handling & Debugging

Proper error handling is essential for creating robust applications that can gracefully handle API failures and unexpected responses.

Common API Errors

// Network errors - unable to reach the server
// HTTP errors - server responded with error status
// Parsing errors - invalid response format
// Timeout errors - request took too long

Comprehensive Error Handling

async function fetchDataWithRetry(url, options = {}, retries = 3) {
  try {
    const response = await fetch(url, options);
    
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }
    
    const data = await response.json();
    return data;
    
  } catch (error) {
    if (retries > 0) {
      console.log(`Retrying... ${retries} attempts left`);
      await new Promise(resolve => setTimeout(resolve, 1000));
      return fetchDataWithRetry(url, options, retries - 1);
    } else {
      console.error('Failed after multiple attempts:', error);
      throw error;
    }
  }
}
